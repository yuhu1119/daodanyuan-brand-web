# BrandWeb 样式规范（精简版）

本规范用于确保官网与后台在后续迭代中保持一致、专业、可维护。

## 1. 设计变量优先

- 前台统一使用 `css/main.css` 的 `:root` 变量（颜色、字号、间距、圆角、阴影）。
- 后台统一使用 `admin/public/admin.css` 的 `:root` 变量，不直接写散落色值。
- 新增样式前优先复用已有变量，不重复创造新语义变量。

## 2. 组件状态一致

- 所有可交互元素必须有清晰的 `:hover` 与 `:focus-visible` 状态。
- `:focus-visible` 使用高对比描边，避免仅靠颜色变化。
- 禁止去掉焦点样式而不提供替代可视反馈。

## 3. 动效与可访问性

- 动效默认短促（150-250ms），避免花哨过渡。
- 必须兼容 `prefers-reduced-motion: reduce`，减少动画干扰。
- 文案与组件不应依赖动画才能理解。

## 4. 布局与间距

- 页面模块间距使用全局节奏（`--spacing-*`）而非任意像素。
- 新区块优先延续既有栅格与容器宽度，避免“局部风格漂移”。
- 空状态要“隐藏不占位”，避免出现空白断层。

## 5. 安全与渲染

- 后台动态插入 HTML 一律先做转义（如 `escapeHtml`），杜绝 XSS 注入。
- 若仅展示文本，使用 `textContent`，不要直接拼接未经处理的 HTML 字符串。

## 6. 评审检查清单（提交前）

- 是否复用了已有变量？
- 是否包含 hover/focus-visible？
- 小屏和大屏是否都检查过？
- 空值是否隐藏且不破坏布局？
- 动态内容是否全部转义？
